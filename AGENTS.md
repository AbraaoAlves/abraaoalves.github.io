## 🛠 Componentes Chave
- **Tokens & tipografia:** (`src/app/globals.css`) - Sistema de 14 tokens por tema (`:root`/`.dark`), escala fluida (`--t-*`), grain, e todo o CSS de componentes/seções portado do protótipo. Fontes em `src/app/layout.tsx` (Space Grotesk display, Hanken body, JetBrains Mono).
- **Primitivas UI:** (`src/components/ui/`) - `Eyebrow`, `Button`, `SectionHead` (ghost + título display).
- **Hero:** (`src/components/hero.tsx`) - Grid do herói + nome como wordmark ASCII (motor existente em `src/components/asciiart/`, via prop `ripple` + fonte de blocos de sombra). Inclui a linha de **disponibilidade** (`.hero-status`, ponto "ao vivo" pulsante + `hero.availability` do content) — o sinal "now" que diferencia os dois CTAs de conversão. No mobile a linha **quebra em 2 linhas** (track `minmax(0,1fr)` + `.hero-left { min-width:0 }`); não a transforme em `inline-flex` sem quebra ou o grid estoura ~390px.
- **Roteamento i18n por URL:** (`src/app/[lang]/`, `src/lib/i18n.ts`) - Static export, então locale **vive na URL**: tudo sob `/en/*` e `/pt/*` (`generateStaticParams → en/pt`). A raiz `/` (`src/app/page.tsx`) é um **preloader/redirect client-side** (`LocaleRedirect`) que escolhe o locale por `localStorage` → senão `navigator.language` (`pickInitialLocale`), com `noscript` de fallback. SEO: `[lang]/layout.tsx` gera title/description localizados + canonical + `og:locale`; hreflang per-URL (+`x-default`) vai no `sitemap.ts`. Helpers em `i18n.ts`: `extractLocale`, `swapLocale`, `localizeHref`.
- **Loading (conexão lenta):** Existe uma única camada: **`BootSplash`** (`src/components/boot-splash.tsx`), um splash inline pré-JS (logo SVG + "loading…") no `layout.tsx`, com CSS autocontido (tema via `prefers-color-scheme`/`.dark`, hex hardcoded — independe do `globals.css`). Ele é renderizado no HTML estático de **toda** página, cobre o cold-load e só some via `classList` após sair de `/` para uma rota localizada (padrão do `reveal.tsx`); fica escondido sob `<noscript>`. A raiz `/` usa `LocaleRedirect`, que escolhe o locale, faz `router.prefetch(locale)` e redireciona imediatamente para `/en` ou `/pt` sem uma segunda tela de loading. **Por que não pré-carregar a home:** a home é só +3KB sobre `/`, então o ganho de rede lenta veio de enxugar o baseline, não de preload.
- **Seções:** (`src/components/sections/`) - Geo, Work (+migrações), **Proof**, Stack, Mentorship compõem a home (`src/app/[lang]/page.tsx`, slotadas via `sections/index.ts`). **Roots** vive em `/[lang]/about` e o **Lab** em `/[lang]/lab` — nenhum dos dois aparece na home.
- **Proof band:** (`src/components/sections/proof.tsx`) - Faixa de provas/outcomes logo após Work: 4 métricas reais (`content.ts → proof.items`, `{figure, label, href}`), grid 4-col → 2×2 no mobile com hairlines (`.proof`/`.proof-grid`/`.proof-cell`). Itens com `href` linkam pro case via `localizeHref`. Stagger via `Reveal`. **Converte afirmação em número** — a espinha de prova da home.
- **Work → Cases:** (`src/lib/cases.ts`, `src/app/[lang]/work/(cases)/`) - Registry de case studies (`Case = {slug,title,org,year,summary,tag}`) espelhando `posts.ts`. Cada case é `(cases)/<slug>/{page.tsx, content.mdx}`: o **`page.tsx` (server) é dono do `metadata` + JSON-LD**, e importa a prosa de `content.mdx` (Code Hike marca MDX como `use client`, então MDX **não pode** `export const metadata` — daí o split). `(cases)/layout.tsx` é o wrapper de prosa + back-link "← Back to work" localizado via `params.lang`. Itens do Work com `caseSlug` (em `content.ts`) ganham o link `.case-link` "Read the case →" (`work.tsx`). Cases hoje: `smartscout-geospatial` (flagship), `notasocial` (founder). **Ao adicionar:** entry em `cases.ts` + pasta `(cases)/<slug>/` + path em `sitemap.ts` `PAGES` (sem descoberta automática) + `caseSlug` no item do Work.
- **Demo interativa:** (`src/components/point-field-demo.tsx`) - Widget client `<PointFieldDemo />` embutido no case SmartScout: renderiza **100k pontos** num canvas (PRNG mulberry32 seeded, clusters tipo-cidade, Float32Array, plot via ImageData/alpha aditivo), com pan (drag), zoom (wheel) e medidor de FPS. Gated por `IntersectionObserver`; estático sob `prefers-reduced-motion`; cor do tema via `getComputedStyle` + `MutationObserver` pra repintar no toggle. É a **prova de habilidade de dev** (demonstra, não conta) — reusa o espírito do motor ascii sem libs novas.
- **Nav & Footer:** (`src/components/header.tsx`, `footer.tsx`) - Nav fixa (blur ao rolar) com a marca à esquerda e, à direita, o `.nav-tools` = toggle de idioma **`PT | EN`** + botão de menu. Footer-contato (slab + backdrop ASCII do logo + spotlight). Links internos passam por `localizeHref(href, lang)`. **Contato dual-intent:** o footer tem dois `mailto:` que deixam o visitante se autoidentificar — "I'm hiring" vs "I have a project" — cada um com `subject` pré-preenchido (`contact.hireLabel/hireSubject/projectLabel/projectSubject` no `content.ts`, via `encodeURIComponent`). Zero backend (fit static export); upgrade pra Formspree/Web3Forms se quiser form depois.
- **Menu (estilo ettrics.com):** (`header.tsx`) - **Único menu em todos os breakpoints** (não há mais nav inline desktop nem dropdown mobile). O botão de duas linhas vira **×** ao abrir um **overlay full-screen** (`.menu-overlay`, dentro da `.nav` z-50, abaixo dos controles da barra via z-index): links grandes left-aligned com índice `01–05` (itens vêm de `contact.navLinks` — mesma fonte do footer), **hover/foco** mantém o item em tinta cheia e apaga os irmãos (`:hover`/`:focus-within`) enquanto um **mostrador gigante de número** à direita rola para o índice ativo (`--active` via JS; `0` fixo + tira `1–N` deslizante; escondido <680px), reveal escalonado + faixa ©/versão embaixo. Comportamento: trava o scroll (`html.menu-open{overflow:hidden}`), Escape e troca de rota fecham (rota via padrão render-time "previous value", sem effect de setState), foco entra no overlay ao abrir e volta ao botão ao fechar. Labels do botão em `nav.menu`/`nav.close` (`content.ts`). Os links ficam no HTML estático (SEO) e o footer repete a nav para humanos sem-JS.
- **Idioma & tema (sem widget flutuante):** O antigo `floating-controls` foi removido. O **toggle de idioma** (`language-toggle.tsx`, `PT | EN` estilo Akita) vive no **header** (esquerda do menu); o **toggle de tema de 3 vias** (`theme-toggle.tsx`, System/Light/Dark com wipe View-Transition) vive no **footer** (`.foot-theme`, direita, acima da faixa ©/versão), ambos como no ettrics.com. `aria-pressed` gateado em `mounted` (hidratação segura).
- **i18n:** (`src/lib/content.ts` + `src/components/language-provider.tsx`) - Conteúdo bilíngue EN/PT (`CONTENT`, `SOCIALS`, + `meta` por locale) consumido via `useLanguage()`. O provider deriva o `lang` da URL via `usePathname()` (servidor e cliente sempre concordam — sem mismatch); `setLang` navega para o caminho equivalente no outro locale e salva a escolha no `localStorage`. PT-BR em voz natural (não tradução literal).
- **Reveal:** (`src/components/reveal.tsx`) - Animação de scroll baseada em CSS (`.js-anim` gate) — conteúdo visível sem JS/para crawlers.
- **Animações & peso do baseline (SSG):** O `template.tsx` (entrada por rota) é **CSS puro** (`.route-enter` no `globals.css`, transform-only, gated em `.js-anim` + reduced-motion) — **framer-motion foi removido** (era o único consumidor; −38KB gzip em toda rota). O `LenisProvider` faz **lazy-load** do Lenis (`import("lenis")` no efeito), tirando-o do baseline inicial. Resultado: baseline de `/` caiu de ~234KB → ~193KB gzip. Mantenha animações novas em CSS/lazy; evite reintroduzir libs pesadas no caminho crítico.
- **Logo:** (`src/components/icons/logo.tsx`) - Logo geométrica; facetas marcadas `.mark-lo` para a nav monocromática de duas tonalidades.
- **Lab:** (`src/app/[lang]/lab/`) - Página `/[lang]/lab` no design do proto (monocromático, eyebrows, bilíngue via `useLanguage`) listando os posts no padrão `.lab-item` (data / `.ti-wrap`: título + `.ex` excerpt 2-linhas / tag). `tag` e `excerpt` (`{en,pt}`) vivem no registry `posts.ts` (não mais hack `postTags` em `page.tsx`). Cada post é `(posts)/<slug>/{page.tsx, content.mdx}` — **mesmo split server/MDX dos cases** (page.tsx é dono de `metadata` + JSON-LD `articleLd`, importa a prosa de `content.mdx`; MDX+Code Hike não exporta metadata). `(posts)/layout.tsx` localiza o "← Voltar ao Lab" via `params.lang`. Posts: `aurelia-to-react`, `rendering-100k-points`, `the-goto-lesson`. (O conteúdo MDX ainda não é traduzido — mesmo post serve nos dois locales.) **Ao adicionar:** entry em `posts.ts` + pasta + path em `sitemap.ts`.
- **SEO / structured data:** (`src/components/json-ld.tsx`, `src/lib/structured-data.ts`) - `<JsonLd data={}>` (server) injeta `<script type="application/ld+json">` (static-export safe). `personLd` (schema `Person`: jobTitle, `sameAs` ← `SOCIALS`, `knowsLanguage`, `knowsAbout`) entra no `[lang]/layout.tsx`; `articleLd(...)` (`TechArticle`) é montado em cada `page.tsx` de post/case. Canonical+hreflang per-route via `metadata`; sitemap com `en/pt/x-default`.
